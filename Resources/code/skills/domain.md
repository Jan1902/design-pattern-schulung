# Kontext
Dieses Projekt ist Teil einer 1,5 tägigen Präsenzschulung zu Design Patterns bei der adesso SE. Zielgruppe sind Junior Devs und Werkstudenten. Das Projekt stellt ein praktisches Beispiel dar, welches eine komplexe Fertigungsmaschine der Zeiss SMT simuliert bzw. daran angelehnt ist.

# Ziel
Es soll eine einfache Softwaresimulation geschaffen werden, die optisch ansprechend und dadurch praktisch und spielerisch Design Patterns näherbringt. Diese Simulation soll eine Maschine beinhalten, Bauteile auf irgendeine Art und weise verarbeiten kann - und das in mehreren komplexen Schritten. Bestimmte Teile der Logik sind exponiert in Katas, die jeweils Code Smells aufweisen und mithilfe eines geeigneten Design Patterns refactort und erweitert werden sollen.

Zu thematisierende Design Patterns:
- Decorator
- Adapter
- Facade
- Builder
- Factory
- Singleton
- Observer
- Command
- Strategy

# Arbeitsweise
Der Techstack ist .NET 10 und WPF. Die "Engine" darf komplex und aufwändig sein, die exponierten Design Pattern Katas sollten hingegen einfach zu verstehen und übersichtlich sein, aber immer einen deutlichen und vor allem branchen-typisch praxisnahen code smell enthalten.
